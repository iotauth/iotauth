# Heterogeneous C Client and Node.js Server

This example runs a secure SST connection between entities written in different languages:

- `heterogeneous.cClient`: a C client built with the SST C API.
- `heterogeneous.nodeServer`: a Node.js server built with `SecureCommServer`.
- `Auth101`: the Java Auth server that authenticates both entities and distributes session keys.

The example uses `examples/heterogeneous/heterogeneous.graph` so it can be generated independently of the larger default graph.

## Repository layout

| Path | Purpose |
| --- | --- |
| `examples/heterogeneous/heterogeneous.graph` | Minimal graph for this scenario. |
| `examples/heterogeneous/heterogeneous.policy.json` | Communication policy that lets the C client request keys for the Node.js server group. |
| `entity/node/example_entities/heterogeneous_server.js` | Node.js secure server. |
| `entity/c/examples/heterogeneous_client/` | C client source, config, and CMake build file. |

## 1. Generate credentials, configs, and Auth database

From `$SST_ROOT/examples`:

```bash
./cleanAll.sh
./generateAll.sh -g heterogeneous/heterogeneous.graph -po heterogeneous/heterogeneous.policy.json
```

In a non-interactive shell or CI run, pass a temporary credential password explicitly:

```bash
./generateAll.sh -g heterogeneous/heterogeneous.graph -po heterogeneous/heterogeneous.policy.json -p testpass
```

This creates:

- Auth properties and database for Auth101.
- Node config at `entity/node/example_entities/configs/heterogeneous/nodeServer.config`.
- Credentials for both `heterogeneous.cClient` and `heterogeneous.nodeServer`.

The C client config is checked in at:

```text
entity/c/examples/heterogeneous_client/heterogeneous_c_client.config
```

It points to the credentials generated by the graph and to the Node server at `127.0.0.1:24100`.

## 2. Build Auth

From `$SST_ROOT/auth/auth-server`:

```bash
mvn clean install
```

## 3. Build the C client

From `$SST_ROOT/entity/c/examples/heterogeneous_client`:

```bash
mkdir -p build
cd build
cmake ../
make
```

## 4. Run the example

Use three terminals.

### Terminal 1: Auth

```bash
cd $SST_ROOT/auth/auth-server
java -jar target/auth-server-jar-with-dependencies.jar -p ../properties/exampleAuth101.properties
```

Auth is ready when it prints:

```text
Enter command (e.g., show re/cp/ta/sk, clean sk):
```

### Terminal 2: Node.js server

```bash
cd $SST_ROOT/entity/node/example_entities
node heterogeneous_server.js configs/heterogeneous/nodeServer.config
```

### Terminal 3: C client

```bash
cd $SST_ROOT/entity/c/examples/heterogeneous_client/build
./heterogeneous_c_client ../heterogeneous_c_client.config
```

You can pass a custom message:

```bash
./heterogeneous_c_client ../heterogeneous_c_client.config "temperature=72 humidity=41"
```

## Convenience scripts

After generating configs and building Auth/C client, these scripts can start each runtime component:

```bash
bash examples/heterogeneous/run_auth101.sh
bash examples/heterogeneous/run_node_server.sh
bash examples/heterogeneous/run_c_client.sh
```

Pass a custom C client message through the client script:

```bash
bash examples/heterogeneous/run_c_client.sh "hello from the C device"
```

## Expected behavior

1. The Node.js server reports that it is listening on port `24100`.
2. The C client requests session keys from Auth101.
3. The C client connects to the Node.js server and completes the SST secure handshake.
4. The C client sends an encrypted message.
5. The Node.js server prints the decrypted message and sends an encrypted acknowledgment.
6. The C client receive thread prints the acknowledgment.

## Troubleshooting

- If the C client cannot find credentials, regenerate with `./generateAll.sh -g heterogeneous/heterogeneous.graph -po heterogeneous/heterogeneous.policy.json` and run the client from the `build` directory.
- If the C client cannot connect, confirm the Node.js server is running and listening on `24100`.
- If Auth rejects the session key request, confirm `exampleAuth101.properties` and the Auth database were generated from `heterogeneous.graph`.
- If Node.js cannot find modules, run `npm install` from `entity/node`.
